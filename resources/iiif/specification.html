IIIF Endpoint

Registration API

Endpoint for institutions to register their content and make it searchable

  • POST iiif/register
  • application/json

Changelog

Date Description
27.05.2019 Initial specification

Request Fields

  • institution String
    Institution in which the content resides.
  • content JSONArray
    Objects that represent a IIIF resource with the following fields:
    • baseURI String
      The IIIF baseURI of the resource
    • collection String
      What collection the image belongs to.
    • manifest String
      Optional: URI of manifest for result presentation.

Example Request

            
        {
          "institution": "Some museum",
          "content": [
            {
              "baseURI": "https://server.tld/iiif/monaLisa.ptif",
              "collection": "Collection Highlights""
            }
          ]
        }

Example Responses

Successful Registration

            
        {
          "status": "200",
          "institution": "National Gallery of Art",
          "numberOfResources":25, 
          "processID": "1"
        }

Missing values, empty fields

            
        {
          "status": "400", 
          "errorMessage": [
            "empty value for institution",
            "missing value for baseURI in element 7",
            "missing value for collection in element 42"
          ]
        }
